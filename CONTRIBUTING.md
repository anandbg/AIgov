# CONTRIBUTING.md

Thanks for considering a contribution. The bar is high on tone, citation discipline, and zero-tracking. The rules below are non-negotiable; CI gates enforce most of them.

## Voice & Style

- **Plain, calm, factual.** No marketing words. No hyperbole.
- **Sentence case** for buttons, headings, and tags.
- **No emojis** in chrome copy. Inline content emoji is reserved for narrative humanisation — never decorate UI controls.
- **Forbidden language** in merged content: `you should`, `you must`, `you are required to`, `this means you have to`. Pitfall 2 — unauthorised practice of law framing. CI grep gate enforces in Phase 2 (CNT-10).
- Cite every regulatory claim. Quotes use the `<RegQuote>` component (Pitfall 6 — the ONLY citation primitive; free-text quoted regulation language is rejected by CI in Phase 2).

## Anti-Features (Will Be Rejected)

The following are explicitly out of scope for v1 and any PR adding them will be closed:

- Google Analytics, Plausible, Fathom, Cloudflare-Web-Analytics-with-cookies, or any tracking script that retains PII.
- Web fonts of any kind. System fonts only (UI-SPEC enforced).
- Cookie consent banners. There are no cookies to consent to.
- Social-share buttons.
- Comment systems on stage pages.
- AI-persona assistant with a name or avatar.
- Lead-generation forms, email captures, newsletter sign-ups.
- Auto-merging AI-drafted PRs. Every PR ships through a human review.

## Content Workflow (Phase 2 onward)

1. **Stage edits** — edit `apps/site/src/content/docs/stages/<stage-slug>.mdx`. Use `<PersonaSection persona="…">` for persona-tagged blocks; quotes via `<RegQuote source="…" article="…" snapshot="…">`; glossary tooltips via `<GlossaryTerm term="…">` (Phase 2 build-time wrapper handles first-occurrence-only behavior).
2. **Glossary entries** — add `apps/site/src/content/glossary/<slug>.md`. Frontmatter must validate against the `GlossaryFrontmatter` schema in `packages/shared/src/schemas/glossary.ts` (definition ≤320 chars).
3. **Regulation snapshots** — leave these to the pipeline (Phase 3 ships `apps/pipeline`). Don't author them by hand unless backfilling a missed scrape, and even then prefer running the pipeline locally.
4. **New fictional company stories** — add `apps/site/src/content/stories/<slug>.md` (Phase 2 CNT-04).

## Pull Request Conventions

Every PR — human or AI-drafted — must include:

- **5-line summary**: source, what changed, why it matters, classification (editorial / clarification / amendment / new-section), risk note.
- **Reviewer checklist** (auto-generated by pipeline PRs; for human PRs use the GitHub PR template).
- For pipeline PRs: a `verify.json` sidecar that lists every cited snapshot URL. CI re-fetches each on review and fails on mismatch (Pitfall 3 mitigation).

Conventional commit prefixes: `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `test:`, `ci:`. Scope the prefix with the plan ID when applicable (e.g., `feat(03-04): ...`).

## Local Development

See [SETUP.md](./SETUP.md) under **Prerequisites** and **Local Development** for the full bootstrap. Briefly:

```bash
nvm use && corepack enable && corepack prepare pnpm@latest --activate
pnpm install
pnpm --filter @aigov/site dev   # → http://localhost:4321
pnpm --filter @aigov/site typecheck
pnpm --filter @aigov/site build
```

## License & Attribution

License terms are finalised in Phase 6 (TAD-01) once the lawyer review completes. Until then, treat all content as "all rights reserved, planning artifact." Cited regulation text remains the property of its issuing authority — every `<RegQuote>` links to the canonical source.
