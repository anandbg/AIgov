# Phase 1 — UI Design Contract

> Visual and interaction contracts for the Foundation & Chrome phase. This phase ships the shell — navigation chrome, color/typography tokens, component contracts, disclaimer surface, and operational UI hooks. Content authoring happens in Phase 2 against these contracts. Generated by gsd-ui-researcher; verified by gsd-ui-checker.

**Audience:** mixed — executives, engineers, compliance officers. Surface persona switcher as a first-class chrome element. **Walks-the-talk constraint:** WCAG 2.2 AA baseline (this is a governance site about responsible AI; the site itself must be exemplary).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro + Starlight, not React/Next/Vite — shadcn not applicable) |
| Preset | not applicable |
| Component library | Starlight 0.39+ (built-in: sidebar, nav, search modal, TOC, dark mode toggle) |
| Custom shell components | Astro components in `apps/site/src/components/` (`<PersonaSwitch>`, `<PersonaSection>`, `<RegQuote>`, `<JurisdictionLens>`, `<DiffViewer>`, `<ChangeLog>`, `<GlossaryTerm>`, `<MermaidJourney>`, `<DisclaimerBanner>`, `<SustainabilityNotice>`) |
| Icon library | **Astro Icon (`astro-icon`) + `@iconify-json/lucide`** — Lucide is the icon family. Bundled at build time; zero runtime JS. Reserve a small fixed set (see Icon Inventory below). |
| Font | **System UI body + system mono code.** No web-font fetches in v1. CSS variables: `--sl-font: -apple-system, BlinkMacSystemFont, "Segoe UI Variable Text", "Segoe UI", system-ui, "Liberation Sans", Helvetica, Arial, sans-serif;` and `--sl-font-mono: ui-monospace, "SF Mono", "Cascadia Code", Menlo, Consolas, "Liberation Mono", monospace;` |
| Theme tooling | Tailwind v4 `@import "tailwindcss"` + `@theme` override block in `apps/site/src/styles/global.css`. Starlight CSS variables (`--sl-*`) are the cross-component contract. |
| Dark mode | First-class. Toggle in Starlight header. Persists via `data-theme="dark|light"` on `<html>`. **System default is `auto`** (respects `prefers-color-scheme`). |
| Density | Comfortable — Starlight defaults preserved: `--sl-content-pad-x: 1rem` (1.5rem at ≥72rem), `--sl-content-width: 45rem` (optimized for 1,500–3,500-word stage explainers). |

**Source:** CONTEXT.md Visual Shell & Component Defaults section (4 locked decisions); REQUIREMENTS.md FND-05.

## Spacing Scale

Declared values (multiples of 4 only). Aliases are Tailwind v4 utilities, which by default map `1` to `0.25rem` (4px) — these align to the 8-point scale we adopt for layout decisions while keeping Tailwind's 1=4px increments available for fine-tuning.

| Token | Value | Tailwind alias | Usage |
|-------|-------|---------------|-------|
| `xs` | 4px | `p-1`, `gap-1` | Icon-to-label gap, tooltip inner padding |
| `sm` | 8px | `p-2`, `gap-2` | Compact element spacing, inline pill internal padding |
| `md` | 16px | `p-4`, `gap-4` | Default element spacing, card internal padding, sidebar item vertical rhythm |
| `lg` | 24px | `p-6`, `gap-6` | Section padding, banner internal padding |
| `xl` | 32px | `p-8`, `gap-8` | Layout gaps between major content blocks |
| `2xl` | 48px | `p-12`, `gap-12` | Major section breaks (between H2 sections in long-form) |
| `3xl` | 64px | `p-16`, `gap-16` | Page-level vertical rhythm (above hero/below footer) |

### Layout-level spacing tokens

| Token | Value | Source | Usage |
|-------|-------|--------|-------|
| Content max-width | `45rem` (720px) | `--sl-content-width` (Starlight default) | Body text column on stage/regulation pages |
| Sidebar width | `18.75rem` (300px) | `--sl-sidebar-width` (Starlight default) | 12-stage spine navigation |
| Nav height | `3.5rem` desktop / `4rem` ≥800px | `--sl-nav-height` | Top chrome (logo + search + persona switch + theme toggle) |
| Content pad-x mobile | `1rem` (16px) | `--sl-content-pad-x` | Horizontal padding under 72rem viewport |
| Content pad-x desktop | `1.5rem` (24px) | `--sl-content-pad-x` ≥72rem | Horizontal padding at desktop |
| Page max-width | `87.5rem` (1400px) | layout shell | Hard cap on overall layout width |

### Exceptions

| Element | Value | Justification |
|---------|-------|---------------|
| Touch targets (icon-only buttons: theme toggle, dismiss, mobile menu, persona pill on mobile) | **min `44 x 44px`** (`min-h-11 min-w-11`) | WCAG 2.2 AA Target Size (Minimum) 2.5.8. Visible 24px icon centered inside 44px hit area. |
| Sticky disclaimer banner inner padding | `12px` vertical × `16px` horizontal (`py-3 px-4`) | Compactness — the banner must not crowd the regulation page header. 12px is a documented exception to 8-point. |
| Focus ring offset | `2px` | WCAG 2.4.11 Focus Not Obscured. Standard accessibility offset. |

## Typography

**Stack:** System UI for body, system monospace for code. **Zero web-font fetches.** This preserves the LCP/INP budget and is locked in CONTEXT.md.

### Type scale (4 sizes declared, mapped to Starlight tokens)

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Body | `--sl-text-base` | **16px** (1rem) | **400 (regular)** | **1.75** (`--sl-line-height`) | Default prose, sidebar items, persona section body |
| Label / small | `--sl-text-sm` | **14px** (0.875rem) | **400** | **1.5** | Metadata (last-scraped date, snapshot date), captions, footer text, breadcrumbs |
| Heading | `--sl-text-h3` / `--sl-text-h2` | **20px** / **24px** | **600 (semibold)** | **1.2** (`--sl-line-height-headings`) | Section headings inside long-form content |
| Display | `--sl-text-h1` | **32px** (2rem) | **600 (semibold)** | **1.2** | Page titles (stage names, regulation overview titles) |

**Weights declared:** exactly two — **400 (regular)** for body/labels and **600 (semibold)** for headings/CTAs. No bold (700) in v1.

**Line heights declared:** exactly two — **1.75** for body (Starlight default, deliberately generous for long-form regulation prose) and **1.2** for headings.

### Code & monospace

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Inline code | **15px** (0.9375rem) | 400 | matches surrounding line | `<code>` inside prose — slightly under body so it sits visually flush |
| Code block | **14px** (0.875rem) | 400 | **1.5** | `<pre>` for engineer-lens code samples; Starlight syntax-highlight defaults |

### Mobile font-size floor

**Minimum 16px for body on mobile** (prevents iOS Safari auto-zoom on form fields, and matches WCAG 1.4.4 text resize). Verified manually on real devices per Pitfall m-3 (PITFALLS.md).

### Forbidden in v1

- No web-font `@import` or `<link rel="preload" as="font">` directives.
- No `font-display: swap` workarounds (because there are no custom fonts).
- No `font-weight: 300` or `100` (thin weights illegible against light/dark backgrounds at body size).
- No `font-style: italic` for emphasis on more than ~5 words at a time (system italics render unevenly across OSes); reserve italics for the Acme Robotics story vignettes (CNT-03, Phase 2 — chrome must support the contract).

**Source:** CONTEXT.md Typography decision; REQUIREMENTS.md FND-02, FND-11; PITFALLS.md m-3; Starlight `props.css` defaults (verified 2026-05-24).

## Color

**Strategy:** Override Starlight defaults via Tailwind v4 `@theme` in `global.css`. Use HSL so the same hue can express `low`/`base`/`high` shades cleanly. Accent is **slate-indigo** (formal but warm) per CONTEXT.md.

### 60 / 30 / 10 split

| Role | Light token | Dark token | Hex (light) | Hex (dark) | Usage |
|------|-------------|------------|-------------|------------|-------|
| **Dominant (60%)** — surface | `--sl-color-bg` | `--sl-color-bg` | `#ffffff` | `#0f1419` (hsl(216, 25%, 8%)) | Page background, content surface |
| **Secondary (30%)** — sidebar, nav, cards | `--sl-color-bg-sidebar`, `--sl-color-bg-nav` | same | `#f6f7f9` (hsl(220, 15%, 97%)) | `#1a1f2b` (hsl(220, 22%, 14%)) | Sidebar, top nav strip, banner backgrounds, RegQuote card surface, ChangeLog item surface |
| **Accent (10%)** — slate-indigo | `--sl-color-accent` | `--sl-color-accent` | `#4f46e5` (hsl(243, 75%, 59%)) | `#818cf8` (hsl(234, 89%, 74%)) | **Reserved — see list below** |
| **Accent low** | `--sl-color-accent-low` | `--sl-color-accent-low` | `#eef2ff` (hsl(226, 100%, 96%)) | `#1e1b4b` (hsl(244, 47%, 20%)) | Active-state backgrounds (current sidebar item, focused persona pill) |
| **Accent high** | `--sl-color-accent-high` | `--sl-color-accent-high` | `#3730a3` (hsl(243, 47%, 42%)) | `#c7d2fe` (hsl(231, 96%, 89%)) | Accent on hover/active text |

### Text colors

| Role | Light | Dark | Contrast vs surface |
|------|-------|------|--------------------|
| Body text | `--sl-color-text` → `#1f2937` (hsl(220, 22%, 17%)) | `#e5e7eb` (hsl(220, 14%, 90%)) | **≥ 12:1** (exceeds WCAG AA 4.5:1) |
| Secondary text | `--sl-color-text-accent` → `#374151` (hsl(220, 19%, 27%)) | `#d1d5db` (hsl(220, 14%, 83%)) | **≥ 8:1** |
| Muted text (metadata, last-scraped) | `--sl-color-text-muted` → `#4b5563` (hsl(220, 14%, 35%)) | `#9ca3af` (hsl(220, 9%, 65%)) | **≥ 4.5:1** (WCAG AA) |
| Heading text | `--sl-color-white` → `#0f1419` light / `#ffffff` dark | inverse | **≥ 14:1** |

### Semantic colors

| Role | Light | Dark | Usage (this phase) |
|------|-------|------|--------------------|
| **Destructive** | `#b91c1c` (hsl(0, 74%, 42%)) | `#f87171` (hsl(0, 91%, 71%)) | Heartbeat-failure issue badge, "scraper drift suspected" PR banner (Phase 3), 4xx/5xx error states. Reserved — never used as decoration. |
| **Warning** | `#b45309` (hsl(31, 88%, 36%)) | `#fbbf24` (hsl(45, 96%, 56%)) | Sustainability mode banner ("maintained at low cadence"), index-updating banner (Phase 5). |
| **Success** | `#15803d` (hsl(142, 71%, 29%)) | `#4ade80` (hsl(142, 76%, 58%)) | "Snapshot verified" badge, axe/Lighthouse pass indicators. Used sparingly; never the default state. |
| **Info / neutral** | `#0369a1` (hsl(202, 94%, 32%)) | `#38bdf8` (hsl(199, 92%, 60%)) | "Not legal advice" pill background — distinct from destructive so it reads as informational, not alarming. |

### Accent reserved for (explicit list — never "all interactive elements")

The 10% accent (`--sl-color-accent`) appears ONLY in:

1. **Active sidebar item** background (`--sl-color-accent-low`) + left rail (3px solid `--sl-color-accent`)
2. **Active persona pill** in the Exec / Engineer / Compliance switcher (background `--sl-color-accent-low`, text `--sl-color-accent-high`, 1px ring `--sl-color-accent`)
3. **Primary CTA** buttons: the "Open search" affordance (⌘K hint), the "Ask in chat" handoff (Phase 5 wiring stub), the persona "Reset to all lenses" link
4. **Inline link text** on hover/focus (default link is `--sl-color-text-accent` → `--sl-color-accent-high` on hover)
5. **Focus ring** for keyboard navigation (`outline: 2px solid var(--sl-color-accent); outline-offset: 2px;`)
6. **Current-page indicator** in breadcrumb/TOC
7. **Mermaid journey diagram**: "current stage" node fill (other stages use neutral grays)

**Not accent (deliberately):**

- Plain body links (use `--sl-color-text-accent`, not `--sl-color-accent`)
- Card borders (use `--sl-color-gray-5` for separation)
- Disclaimer banner background (uses neutral `--sl-color-bg-nav` so it doesn't compete with regulation content)
- "Not legal advice" pill (uses Info color so it reads as a fact-statement, not a feature highlight)
- Mermaid edges (default neutral, accent only on the "you are here" arc)

### Contrast verification

Every text-on-surface combination above is **manually verified against WCAG 2.2 AA (4.5:1 body, 3:1 large text)** and target ≥ AA Large (7:1) where feasible. The axe-core CI gate (FND-10) is the enforcement layer.

**Source:** CONTEXT.md color decision (slate-indigo accent); REQUIREMENTS.md FND-11 (WCAG 2.2 AA), FND-14 (disclaimer surface); PITFALLS.md Pitfall 2 (disclaimer visibility), m-3 (a11y).

## Motion & Interaction

### Motion tokens

| Token | Value | Usage |
|-------|-------|-------|
| `--motion-fast` | `120ms` cubic-bezier(0.2, 0, 0, 1) | Hover state on links, persona pill, sidebar item |
| `--motion-base` | `200ms` cubic-bezier(0.2, 0, 0, 1) | Theme toggle, banner appear/dismiss, mobile sidebar slide |
| `--motion-slow` | `320ms` cubic-bezier(0.2, 0, 0, 1) | Disclaimer banner first-time reveal (only on regulation pages) |

### `prefers-reduced-motion`

**Default behavior is reduced motion.** Honor the user preference at the CSS layer:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This is non-negotiable per WCAG 2.3.3 Animation from Interactions.

### Interaction contracts

| Element | Trigger | Behavior |
|---------|---------|----------|
| Persona pill toggle | click / Enter / Space | Sets `data-persona` on `<html>`, updates URL to `?lens=<persona>` (no hash, so search engines treat it as same page); persists choice in `localStorage` key `aigov:persona` |
| Persona "Reset" link | click | Sets `data-persona="all"`, removes `?lens=` query, clears localStorage entry |
| Sidebar item | hover / focus | Background → `--sl-color-gray-6` (light) / `--sl-color-gray-5` (dark); 120ms |
| Sidebar item active | current page | Background → `--sl-color-accent-low`; left rail 3px `--sl-color-accent`; text weight 600 |
| Search modal | ⌘K (macOS) / Ctrl+K (Win/Linux) / click search icon | Opens Pagefind modal (Starlight built-in); ESC closes; results focusable by arrow keys |
| Theme toggle | click | Cycles `auto → light → dark → auto`; aria-label announces current state |
| Disclaimer banner (per-regulation page only) | first visit per session | Slides down (320ms); dismiss button at right; reappears on navigation to a different regulation page |
| Mobile sidebar | hamburger tap | Slides in from left (200ms); ESC or backdrop tap dismisses; trap focus inside while open |
| GlossaryTerm hover/focus | hover or keyboard focus | Tooltip pops above term (≤320 char definition + "open glossary →" link); ESC dismisses; first-occurrence-per-page only |
| Heartbeat-failure issue badge (Maintainer view, About page) | always visible if state.heartbeat = failing | Solid destructive background + bell icon; 0 animation (errors are static) |
| Sustainability-mode notice banner | flag in `site.ts` is `true` | Renders at top of every page below nav; warning-color background; non-dismissible; persistent text "Maintained at low cadence — last reviewed YYYY-MM-DD"; cycle of low-cadence cron schedule listed in About page |

### Focus management

- **Keyboard focus ring:** `outline: 2px solid var(--sl-color-accent); outline-offset: 2px; border-radius: 4px;` on EVERY focusable element. Never `outline: none` without a replacement (WCAG 2.4.7).
- **Tab order:** Skip-to-content link is the first focusable element on every page (`<a class="skip-link" href="#main-content">Skip to content</a>`); visible on focus only.
- **Modal traps:** Search modal and mobile sidebar trap focus inside while open. Tested with axe-core.

## Component Contracts

These are the **shell** contracts. Phase 2 fills them with content. Each contract specifies what executor must build in Phase 1 (FND-05).

### `<PersonaSwitch>` (sticky chrome element)

| Property | Value |
|----------|-------|
| Location | Right side of Starlight nav (between search and theme toggle), sticky with `position: sticky; top: 0;` |
| Visual | Three-segment pill: `[ Exec ][ Engineer ][ Compliance ]` + optional "All" reset link below pill on mobile |
| Active state | Active segment: `--sl-color-accent-low` bg, `--sl-color-accent-high` text, 1px `--sl-color-accent` ring; inactive: `--sl-color-bg-nav` bg, `--sl-color-text-accent` text |
| Default lens | "All" (all three sections visible). Reading `?lens=`, then `localStorage`, then "all" if neither set. |
| Mobile (<768px) | Collapses to icon + label "Lens: All ▾"; tap opens bottom sheet with the three options + Reset |
| Accessibility | `role="tablist"`, each pill `role="tab"` with `aria-selected`, `aria-controls` pointing to body; full keyboard nav (← → arrows cycle, Home/End jump) |
| URL contract | `?lens=exec` / `?lens=engineer` / `?lens=compliance` / `?lens=all` (or no param = all). Anchor preservation: query is set *before* hash so `#section` still works. |
| JS cost | ≤ 1KB inline script that flips `data-persona` on `<html>`; CSS does the rest. No hydration of a framework component. |

| Property | Value |
|----------|-------|
| Render | `<section data-for-persona="{persona}" class="persona-section persona-{persona}">` — always in DOM (for SEO + Pagefind indexing) |

| Style | Left border 3px in persona-tinted color (subtle: 30% saturation of accent for exec, neutral gray for engineer, semantic info for compliance — finalized in CSS but kept distinct enough to be identifiable in "all" mode); padding `16px` left, `8px` vertical |
| Label | Discreet persona tag in top-right of section: `EXEC` / `ENGINEER` / `COMPLIANCE` — 12px caps, letter-spacing 0.05em, color `--sl-color-text-muted` |
| Print | All persona sections render in print regardless of `data-persona` (print stylesheet sets `display: block !important` on `[data-for-persona]`) |
| No-JS fallback | Works without JS: default `data-persona` is "all", so all sections render. Toggle is a progressive enhancement. |

### Destructive actions in Phase 1

Phase 1 has **no destructive actions** (no delete, no irreversible toggles, no data writes from the UI). The disclaimer banner dismiss is per-session only — not destructive.

If a future phase introduces a destructive action, the confirmation pattern is:
- Modal title: `{verb}: {noun}?` (e.g., "Delete bookmark?")
- Body: explicit, factual ("This removes the bookmark from this browser. There is no undo.")
- Primary button: destructive color, verb-noun label ("Delete bookmark")
- Secondary button: neutral, "Cancel"
- Focus default: Cancel (not the destructive action)

## Accessibility Contract (WCAG 2.2 AA)

This is a governance site — accessibility is part of the trust story, not optional polish.

### Required at Phase 1 (verified by axe-core CI per FND-10)

| Criterion | Implementation |
|-----------|----------------|
| 1.3.1 Info & Relationships | Semantic HTML: `<nav>`, `<main>`, `<aside>`, `<article>`, `<footer>`. ARIA only when semantic HTML insufficient (e.g., persona pills as `role="tablist"`). |
| 1.4.3 Contrast (Minimum) | All text-on-surface combinations ≥ 4.5:1 (≥ 3:1 for large text ≥18pt or ≥14pt bold). Verified above. |
| 1.4.4 Resize Text | Text scales to 200% without horizontal scroll on a 320px viewport. Use `rem`/`em` for font sizes; never `px`. |
| 1.4.10 Reflow | 320px viewport works without horizontal scroll. Tested on real iPhone SE width. |
| 1.4.11 Non-text Contrast | Icons, focus rings, persona-pill borders all ≥ 3:1 against adjacent colors. |
| 1.4.12 Text Spacing | No `line-height` < 1.5 on body text; no `letter-spacing` < 0.12× font size. |
| 1.4.13 Content on Hover or Focus | Tooltips (glossary, citation hover cards) are dismissible (ESC), hoverable (cursor can enter without dismissing), and persistent (don't disappear before the user can read). |
| 2.1.1 Keyboard | Every interactive element reachable and operable by keyboard. No keyboard traps (search modal, mobile sidebar have explicit traps with ESC release). |
| 2.4.3 Focus Order | Tab order matches visual order. Skip-to-content link is first. |
| 2.4.4 Link Purpose | Link text describes destination ("View source on GitHub", not "click here"). |
| 2.4.7 Focus Visible | Every focusable element shows a 2px outline on focus. Never `outline: none` without replacement. |
| 2.4.11 Focus Not Obscured | Sticky banner has `top: var(--sl-nav-height)` so focused elements aren't hidden under chrome. |
| 2.5.8 Target Size (Minimum) | All interactive icon-only controls (theme toggle, dismiss, mobile menu) are at least 44×44 CSS pixels. |
| 3.1.1 Language of Page | `<html lang="en">` set explicitly (i18n deferred per AF-4). |
| 3.2.3 Consistent Navigation | Nav order is identical on every page. |
| 3.3.2 Labels or Instructions | Form-like controls (search input) have visible labels. |
| 4.1.2 Name, Role, Value | Custom widgets (persona switcher, glossary tooltip, theme toggle) expose accessible name/role/state via ARIA. |

### Out of scope for Phase 1 (lands in later phases)

- 2.4.5 Multiple Ways — depends on full nav (Phase 2 spine) + search index density (Phase 2 content). Chrome supports it.
- 1.2.* Time-based Media — no video/audio in v1.
